# Task 2: プロジェクトでClaude Codeをアクティベートする

## 目的
このトレーニングプロジェクト（example1ディレクトリのNext.jsプロジェクト）でClaude Codeを使えるようにする

## 所要時間
約10分

## 前提条件
- Task 1が完了していること
- Claude Codeがインストールされていること

## 手順

### 1. プロジェクトディレクトリに移動

```bash
cd /path/to/claude-code-training/example1
```

### 2. Claude Codeを起動

```bash
claude
```

### 3. プロジェクトメモリ（CLAUDE.md）を作成

プロジェクト固有の指示やルールを保存するため、CLAUDE.mdファイルを作成します。

#### 方法1: `/init`コマンドを使用（推奨）

Claude Codeのチャット画面で以下のコマンドを入力します：

```
/init
```

これにより、CLAUDE.mdファイルの作成がガイドされます。

#### 方法2: `/memory`コマンドを使用

```
/memory
```

このコマンドで、インタラクティブにメモリを編集できます。

#### 方法3: 手動で作成

以下のいずれかの場所にファイルを作成します：

- `CLAUDE.md` （プロジェクトルート）
- `.claude/CLAUDE.md` （.claudeディレクトリ内）

どちらでも機能します。

### 4. CLAUDE.mdの内容を理解する

CLAUDE.mdファイルには以下を記述できます：

- **プロジェクト固有の指示**: 「このプロジェクトはNext.js 15を使用」
- **コーディング規約**: 「TypeScriptを使用し、関数コンポーネントで記述」
- **カスタムルール**: 「コメントは日本語で記述」
- **技術スタック**: 使用しているライブラリやフレームワーク

### 5. CLAUDE.mdをカスタマイズ（オプション）

例えば、以下のような指示を追加できます：

```markdown
# このプロジェクトについて

このプロジェクトはNext.js 15を使用した学習用プロジェクトです。

## コーディング規約

- TypeScriptを使用
- コンポーネントは関数コンポーネントで記述
- コメントは日本語で記述
```

### 6. 設定の確認

Claude Codeで以下のように質問してみましょう：

```
このプロジェクトの構成を教えてください
```

Claude Codeがプロジェクトのファイル構造を理解し、適切に回答してくれるはずです。

## 確認事項

- [ ] CLAUDE.mdファイルが作成されている（ルートまたは.claudeディレクトリ内）
- [ ] CLAUDE.mdにプロジェクトの情報を記述した
- [ ] Claude Codeがプロジェクトのコンテキストを理解している

## CLAUDE.mdとは？

`CLAUDE.md`は、Claude Codeにプロジェクト固有の指示を与えるためのメモリファイルです。

### メリット
- プロジェクトごとに異なるコーディング規約を設定できる
- チーム全体で共通の指示を共有できる
- Claude Codeの回答品質が向上する
- セッションをまたいで情報が保持される

### 階層的なメモリシステム

Claude Codeは階層的なメモリシステムを持っています（優先順位順）：

1. **Enterprise レベル**: `/Library/Application Support/ClaudeCode/CLAUDE.md` （最優先）
2. **Project レベル**: `./CLAUDE.md` または `./.claude/CLAUDE.md`
3. **User レベル**: `~/.claude/CLAUDE.md`

上位の階層が優先されます。つまり、プロジェクト固有の設定はユーザーレベルの設定を上書きします。

### ファイルの配置場所

プロジェクトレベルでは以下の2つの場所が使用できます：

- `CLAUDE.md` - プロジェクトルートに直接配置
- `.claude/CLAUDE.md` - .claudeディレクトリ内に配置

**推奨**: `.claude/CLAUDE.md` を使用すると、他の設定ファイル（commands、agents）と一緒に管理できます。

## 便利な機能

### クイック追加: `#` ショートカット

会話中に`#`で始まるメッセージを送ると、自動的にCLAUDE.mdに追加されます：

```
# このプロジェクトではTailwind CSSを使用しています
```

### ファイルのインポート

CLAUDE.mdから他のファイルをインポートできます：

```markdown
# プロジェクト設定

@docs/coding-standards.md
@docs/architecture.md
```

最大5階層まで再帰的にインポート可能です。

### ベストプラクティス

1. **具体的に記述**: 曖昧な指示は避け、具体的に
2. **構造化されたマークダウン**: 見出しと箇条書きを使用
3. **定期的に見直し**: プロジェクトの進化に合わせて更新
4. **チームで共有**: Gitにコミットしてチーム全体で使用
