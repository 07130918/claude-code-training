# Task 3: WebSearch機能を有効化する

## 目的
Claude CodeのWebSearch機能を有効化し、最新の情報をWeb検索できるようにする

## 所要時間
約5分

## 前提条件
- Task 1, 2が完了していること
- Claude Codeが起動していること

## WebSearch機能とは？

WebSearch機能を有効にすると、Claude Codeが：
- 最新のライブラリ情報を検索
- ドキュメントを参照
- エラーメッセージの解決方法を調査
- 最新のベストプラクティスを取得

など、リアルタイムの情報にアクセスできるようになります。

## 手順

#### 1. WebSearchを有効化

`.claude/settings.local.json`を以下のように編集します：

```json
{
  "permissions": {
    "allow": ["WebSearch"]
  }
}
```

#### 2. Claude Codeを再起動

設定を反映させるため、Claude Codeを再起動します。

### 3. 動作確認

WebSearchが有効になったか確認するため、Claude Codeに質問してみましょう：

```
Next.js 15の最新機能について教えてください
```

Claude Codeが実際にWeb検索を行い、最新情報を取得している様子が見られるはずです。

## 確認事項

- [ ] Claude CodeがWeb検索を実行できる
- [ ] 検索結果が表示される

## WebSearchの活用例

### 1. 最新ドキュメントの参照
```
TypeScript 5.7の新機能について調べて
```

### 2. エラー解決
```
"Cannot find module" エラーの解決方法を検索して
```

### 3. ベストプラクティスの確認
```
React Server Componentsのベストプラクティスを検索して
```

### 4. ライブラリの比較
```
状態管理ライブラリのZustandとJotaiを比較して
```

## 注意事項

### 検索の制約
- 検索結果の品質は検索クエリの質に依存します
- 一部のサイトはアクセス制限がある場合があります

### プライバシー
- 検索クエリはAnthropicのサーバーを経由します
- 機密情報を含む検索は避けてください

## 次のステップ

✅ WebSearchが有効化できたら、[task4.md](./task4.md)に進んで実際にWebSearchを使ってNext.jsについて学びましょう！

## トラブルシューティング

### WebSearchが動作しない場合
1. 記述が正しいか確認
2. JSON形式が正しいか確認（カンマ、括弧など）
3. Claude Codeを完全に再起動
4. ネットワーク接続を確認

### 検索結果が表示されない場合
- 質問を具体的にする
- 検索キーワードを変えてみる
- より明確な指示を出す（例：「〜を検索して」）
