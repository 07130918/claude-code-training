# コードレビュー

このプロジェクトの最近の変更をレビューしてください。

## レビュー手順

1. **変更の確認**
   - `git status` で変更されたファイルを確認
   - `git diff` でステージされた変更とステージされていない変更を確認
   - `git diff HEAD` ですべての変更を確認

2. **コード品質のチェック**
   - コードの可読性（変数名、関数名、コメント）
   - コードの構造（関数の長さ、複雑度、責務の分離）
   - 重複コードの有無
   - エラーハンドリングの適切性

3. **ベストプラクティスの確認**
   - 言語・フレームワーク固有のベストプラクティスに従っているか
   - デザインパターンの適切な使用
   - パフォーマンスの考慮（不要な計算、メモリ使用）
   - アクセシビリティの考慮（該当する場合）

4. **セキュリティチェック**
   - 機密情報のハードコード（API キー、パスワード等）
   - 入力のバリデーション
   - SQLインジェクション、XSS等の脆弱性
   - 適切な認証・認可の実装

5. **TypeScript/型の確認**（該当する場合）
   - 適切な型定義の使用
   - `any` の不要な使用を避けているか
   - 型安全性の確保

6. **テスト**
   - 既存のテストが通るか確認
   - 新機能に対するテストの必要性
   - テストカバレッジの確認

7. **ドキュメント**
   - コードコメントの適切性
   - README等のドキュメント更新の必要性
   - 複雑なロジックの説明

## レビュー結果の報告

以下の形式でレビュー結果を報告してください：

### ✅ 良い点
- （良い実装や改善点をリストアップ）

### ⚠️ 改善提案
- （改善が推奨される点をリストアップ。重要度と理由を含める）

### 🚨 重大な問題
- （セキュリティやバグなど、修正が必須の問題をリストアップ）

### 💡 オプション提案
- （必須ではないが、検討に値する改善案をリストアップ）

## 注意事項

- 建設的で具体的なフィードバックを提供する
- 問題を指摘するだけでなく、解決策も提案する
- コードスタイルよりも、ロジックやセキュリティを優先する
- プロジェクトのコンテキストを考慮する
